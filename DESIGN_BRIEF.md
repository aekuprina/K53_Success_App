# Design Brief — редизайн K53 Success App

Бриф для дизайн-сессии (Claude design / Figma). Текущая версия — функциональный MVP с utility-дизайном; цель — «значительно выше конкурентов» и по ощущению тоже.

## Продуктовый контекст

- Аудитория: 17–24, ЮАР, бюджетные Android-телефоны, дорогой трафик, английский часто второй язык
- Обещание бренда: «сдал с первого раза», честный прогноз готовности
- Конкуренты выглядят как quiz-фабрики 2015 года — визуальный отрыв сам по себе конверсионный аргумент
- Live: https://k53success.vercel.app · Репо: github.com/aekuprina/K53_Success_App

## Жёсткие ограничения (нарушать нельзя)

1. **Перформанс:** first load < 2 MB, время до первого вопроса < 60 сек на бюджетном Android. Никаких тяжёлых шрифтов-вебфонтов, фоновых видео, lottie-полотен. Сейчас first-load JS ~115 KB — держать порядок.
2. **Стек:** Next.js + Tailwind, один шрифт system-ui. Дизайн-токены в `tailwind.config.ts`.
3. **Офлайн и dark mode** обязаны продолжать работать.
4. **Доступность:** тач-таргеты ≥ 44px, контраст WCAG AA, работа одной рукой (нижняя навигация).
5. Мультиязычность: строки объяснений бывают длиннее английских на ~30% (af/zu/xh) — макеты не должны ломаться.

## Текущая система

- Цвета: brand green #158758 (600) / #22a76e (500), ink-серые, dark = slate
- Радиусы: rounded-2xl карточки, rounded-xl кнопки
- Компоненты: card, btn-primary, btn-ghost, option-btn, ScoreRing (SVG-кольцо), нижняя таб-навигация (5 пунктов)
- Экраны: welcome (онбординг), home (Readiness Score + danger topics + 4 плитки), practice (список тем с прогресс-барами), practice/session (квиз), exam (интро → 68 вопросов с таймером → результаты по блокам), mistakes, signs (грид + поиск + фильтры), signs/[id], settings

## Что хочется от редизайна

1. **Индивидуальность.** Сейчас — «аккуратный generic Tailwind». Нужен характер: ЮАР-контекст без клише, энергия «я это сдам», но не детская геймификация.
2. **Readiness Score как герой.** Это главный дифференциатор — кольцо на главной должно ощущаться как «мой прогноз», достойный шеринга в WhatsApp (подумать про share-карточку).
3. **Экзаменационный режим** — визуально отличим от практики: серьёзнее, ближе к реальному CLLT-интерфейсу (это селлинг-поинт «точная копия теста»).
4. **微-моменты:** правильный/неправильный ответ, окончание серии, рост Score — лёгкая, дешёвая по ресурсам обратная связь (CSS-анимации, не JS-библиотеки).
5. **Пустые состояния** (нет ошибок, нет моков) — сейчас голый текст.
6. **Иконография:** сейчас эмодзи на плитках — заменить на консистентные SVG-иконки.
7. **Signs-страницы** — публичные SEO-лендинги, по ним придёт органика: нужен вид, которому доверяешь, и CTA в приложение.

## Метрики успеха редизайна

Рейтинг в сторах ≥ 4.7; конверсия welcome → первый вопрос без падения (< 60 сек); D30 retention ≥ 20%. Дизайн не должен стоить ни одного из перечисленных KPI перформанса.
