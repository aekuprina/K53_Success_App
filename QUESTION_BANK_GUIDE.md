# Question Bank Guide — брифинг для агента-генератора

Цель: довести банк со 150 до **1200+ вопросов**, батчами по ~100, без дублей и без копирования официальных материалов.

## Формат вопроса

```ts
{ id: "r063", block: "rules", topic: "speed",
  q: "What is the speed limit when towing another vehicle with a rope?",
  options: ["Maximum 30 km/h", "Maximum 60 km/h", "The normal limit for that road"],
  answer: 0,   // индекс правильного ответа (движок перемешивает варианты сам)
  explain: "When towing with a tow-rope you may not go faster than 30 km/h.",
  rule: "NRTA reg. XXX"  // опционально
}
```

## Жёсткие правила

1. **Только собственные формулировки.** Не копировать официальный учебник K53, вопросы DoT/RTMC или конкурентов. Смысл правила — можно, формулировка — своя.
2. **Grade 8 English.** Короткие предложения, простые слова. Целевая аудитория — люди, для которых английский второй язык.
3. **Не повторяться.** Перед добавлением батча прогнать `node scripts/validate-bank.mjs batch.json`. Валидатор ловит точные дубли и похожие вопросы (Jaccard ≥ 0.75) против ВСЕГО существующего банка, включая 150 стартовых.
4. **ID:** префикс по блоку (`r`/`s`/`c`) + следующий свободный номер. Занято: r001–r062, s001–s058, c001–c030.
5. **3 варианта ответа** (стандарт CLLT), 4 допустимо. Дистракторы правдоподобные, не абсурдные.
6. `answer` всегда указывает на правильный вариант; позиция в массиве не важна (движок перемешивает).
7. **Объяснение обязательно:** 1–2 предложения, почему ответ такой, простым языком.
8. Факты сверять с NRTA (Act 93 of 1996) и актуальными правилами. Числа (лимиты, дистанции, сроки) — критичны: 60/100/120 км/ч, 24 месяца learner's, 0.05 г/100мл, 1.5 м гидрант, 5 м перекрёсток, 9 м пешеходный переход, 1 мм протектор, 30 км/ч буксировка.

## Процесс батча

1. Сгенерировать ~100 вопросов как JSON-массив → `batch-XXX.json`
2. `node scripts/validate-bank.mjs batch-XXX.json` → исправить ошибки/варнинги
3. Конвертировать в TS и добавить в новый файл `data/questions-ext-NNN.ts` (тот же формат, `export const EXT_NNN: Question[]`)
4. Подключить в `data/bank.ts`: импорт + добавить в массив `BANK`
5. `npm run build` должен пройти
6. Прогнать валидатор ещё раз без аргументов (весь банк)

## Локализация новых вопросов

Вопросы остаются на английском. Для каждого нового вопроса добавить переводы **объяснения** и **хинта вопроса** в `data/i18n/af.ts`, `zu.ts`, `xh.ts` (формат `id: { q: "...", explain: "..." }`). Если перевод пока не готов — не блокирует: UI автоматически падает обратно на английский.

## Пометки для петли калибровки (v1.1)

Поле `rule` заполнять где возможно — оно станет основой связи «вопрос → тема реального экзамена» при еженедельной перекалибровке весов.
